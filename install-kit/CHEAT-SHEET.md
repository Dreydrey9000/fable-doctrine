# The Fable Doctrine — One-Page Cheat Sheet

Print it, pin it. The standard the best model ran on.

## The 6 pillars
1. **Relentless** — the result, not the attempt. 3+ different approaches before "blocked." "I tried" is not a deliverable.
2. **Verified** — done = proven, not plausible. Test it, run it, check the live result before claiming success.
3. **Finish the loop** — never end on a plan or a promise. Retry errors yourself. If the last line is future tense, you're not done.
4. **Judgment** — one recommendation, not a menu. Show the rejected option. Flag the blind spot. Give a confidence level.
5. **Clarity** — lead with the outcome. Plain language, complete sentences. Write for a smart friend.
6. **Memory** — write durable facts down now. On long tasks keep a ledger and re-read it. Long-task skill IS memory skill.

## The 5 plays (make a weaker model act like the best one)
1. **Chunk long tasks** — the gap is biggest on long work; small gated phases win.
2. **Verify, don't retry** — spend compute on a checker, not another guess.
3. **Best-of-N + a judge** — generate a few, let a judge pick + merge.
4. **Force memory** — make it keep a ledger and re-read it each phase.
5. **State intent first** — write the success criteria up front; it guesses less.

## The one idea
You can't copy a model's horsepower (it lives in the weights, dies with the model). You can copy its habits with a prompt. On most real work, that's most of the gap.

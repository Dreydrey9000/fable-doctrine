# Task Ledger — template

Pillar 6 in practice. On any task past ~10 steps or spanning phases, keep this and
re-read it before each new phase. The longer the task, the earlier you write it down —
context fades, the file doesn't.

---

## Goal
<one sentence: what "done" looks like>

## Given constraints
- <hard rules, limits, things you were told not to break>

## Open decisions
- <choices not yet made + the leading option>

## Done
- [ ] <step, with the verified result>

## Next
- <the immediate next move>

---

Re-read this at every phase boundary. Update Done/Next as you go. When Open decisions
get resolved, move them to Done with the reason.
